# SPPH 381H — Your Personal Workspace

This is **your** copy of the working project for **SPPH 381H: Health Data
Science (AI and Knowledge Translation)**. You created it by **forking** the
course template. You will do your hands-on work here — editing and rendering
Quarto documents, then committing and syncing — and submit your repository link
on Canvas.

> The course book itself lives in a separate, read-only repository. You do not
> edit the book; you work here, in your own fork.

## What is in this project

```
.
├── data/                 <- Practice data. READ-ONLY: import from it, never save into it.
│   └── patients.csv      <- A small SYNTHETIC (fake) dataset for practice.
├── output/               <- Generated files. Your code writes cleaned data and figures here.
├── practice_report.qmd   <- Your first Quarto notebook. Open and render this.
├── README.md             <- This file.
└── .gitignore            <- Tells Git which files to ignore.
```

## Getting started (the full walkthrough is on the Week 2 course page)

1. On this repository's GitHub page, click the green **`< > Code`** button →
   **Codespaces** tab → **Create codespace on main**.
2. Wait a few minutes for the environment to build (R, Quarto, and Python
   install automatically). You will see a **TOOLCHAIN SELF-TEST** in the
   terminal when it finishes.
3. In the file list on the left, open **`practice_report.qmd`**.
4. Click **Render** (top-right of the editor). An HTML preview with a chart
   should appear, with no red error messages.
5. Make a small edit (for example, put your name in the `author` field), render
   again, then **commit and sync** your work.

## Important

- **`data/` is read-only.** Import from it; never save into it.
- **`output/` is generated** by your code.
- The practice dataset is **synthetic** — it is not real patient data. **Never
  put real, private, or personal health data in this repository.**
